**Synthesized 2026-07-13** from two independent drafts and mutual cross-reviews:
- Draft A: `data/plan-fable-r7/report.md` + critique `data/plan-fable-r7/critique.md`

## 1. Settled foundations (unanimous - do not relitigate)

- **No ECS.** Plain data-oriented `GameRuntime`: stable IDs, packed stores, ordered systems. R3F/React is view/lifecycle only; low-rate external UI store; no zustand in the sim.
- **One fixed 60 Hz simulation** driven from a single `useBeforePhysicsStep`, `timeStep={1/60}`, catch-up clamp 100 ms/5 steps with an overrun metric, input-intent ring buffer consumed per tick. (Draft B's "120 Hz" ledger line was a typo both critiques flagged; 120 Hz is only a fallback spike if 60 Hz fails crest/CCD/hover acceptance.)
- **Determinism scope:** same-build local replay only - integer ticks, seeded RNG, stable ordering, quantized inputs, golden snapshot hashes. No lockstep/rollback claims. Plus a body-insertion-order gate: sim registers bodies deterministically ("all racers ready → seed → tick 0"), replay harness asserts the body-order manifest (React/GLTF resolve order is not deterministic on its own).
- **Dynamic Rapier hover craft** (not kinematic): raycast spring/damper suspension, pure craft math returning force commands through a Rapier adapter that owns queries/handles.
- **Energy is one economy**: integer 0-1000 (UI shows %), spent by boost, drained by damage, restored by pits/absorb/kill refunds. Death at zero.
- **Server-authoritative race/combat RULES, client-authoritative movement** - named honestly: *client movement authority with server validation*, casual tier, disclosed in release notes. Never described as prediction/reconciliation.
- **Exact 1-second in-place respawn** (brief requirement): offline tick+60 exactly; online logical deadline `respawnAt = deathAt + 1,000,000 μs` (state transitions are deadline-driven even if the materializing reducer runs late; clients clock-sync and present at `respawnAt`).
- **Bot-driven QA** with the same intent interface as players, from Phase 1.
- **CC0-first assets**; no motion blur ever; readability beats spectacle.

## 3. Gaps neither draft caught alone (all adopted)

Pause semantics (solo = real sim pause; online = presentation-only overlay with LIVE indicator, input-neutral, throttle released). Tab-visibility policy (hidden publisher → `parked` flag, ghosted for others; >30 s hidden while racing → DNF path). Versioned client persistence (`persistence.ts`, schema version + migration: settings/keybinds/best laps/livery). Playtest telemetry = local ring buffer + JSON download in dev builds only. Onboarding: skippable 60-90 s first-run training sequence on Neon Orbital, data-driven, real-binding display ("stranger finishes 3 laps" gate is unreachable without it). Photosensitivity gate separate from reduced-motion (no full-field flicker, flash frequency/luminance-area caps, non-color telegraphs). Server abuse limits (per-identity rate/payload bounds, projectile/mine caps, event TTL, match expiry, rejection counters, malicious-client soak). SpacetimeDB maincloud cost spike (measure reducer/row-update cost at 20 Hz × 8; go/no-go hosting decision in P4.1). Respawn clock-fault policy (reconnect crossing `respawnAt` derives state from server time; never replays stale death animation). Artifact compatibility contract (`{compilerVersion, trackId, trackVersion, gameplayHash, artVersion}`; gameplay-hash change invalidates replays/AI lines/matches; art-only doesn't; CI regenerates and fails on dirty output).

## 5. Roadmap (phases = strings of PR-sized items; critical path in bold)

- **G0 (in flight now):** track visibility + steering inversion + turn smoothing fix. Consumed, not duplicated.
- **N0 - Network containment gate (small, immediately after G0):** close the five live findings before the game advertises multiplayer: (1) reject client-declared respawn discontinuities unless matching a server-issued token/pose; movement limits vs server elapsed time; (2) wire gate sensors → `reportCheckpointCrossed` with monotonic sequence + idempotency + a two-client integration test finishing 3 laps; (3) boot-time mode select - Solo (`LocalMatchAdapter`) vs Online (`SpacetimeMatchAdapter` as sole phase/clock authority, local auto-start suppressed); (4) `request_rematch` (valid only in `finished`, transactional reset, host-confirm or all-request, lowest-slot host election) - never render a Ready the reducer rejects; (5) disconnect policy: lobby/countdown clears ready + cancels <2; racing → ghost + 15 s reconnect grace → DNF (terminal for all-finished logic). Multiplayer UI feature-gated "experimental" until N0's flow test passes.
- **P1 - Time-trial vertical slice (fun, fast):** 60 Hz runtime PR → *quick feel wins PR* → RMF track compiler + hashed artifact → CraftController v2 (airbrakes/sideshift, frequency-parameterized suspension) → camera/readability slice + onboarding + minimal audio + line-bot smoke + `__KZERO_TEST__` hook. **Exit gate: five blind testers finish 3 clean laps and rate control + readability ≥ 4/5.**
- **P2 - Offline combat:** literal-pose respawn (tick+60), two-slot inventory + absorb, pickup rows/sockets, consolidated roster with control-loss caps, telegraphs/counters, energy economy, balance instrumentation + bot duels.
- **P3 - AI + second track:** lateral-line AI + personalities + overtake planner, competition director (bounded, perceptual rules), AI item use; Black Rain Foundry greybox → art. Gate: both tracks × 8 bots pass balance/fairness suites before any network schema freezes.
- **P4 - Multiplayer v1:** pinned SpacetimeDB toolchain + maincloud cost go/no-go; match-scoped schema migration; authoritative lobby/laps/results/pickups/inventory/energy/damage/death/respawn deadlines; movement validation + interpolation (`discontinuityId`, clock sync); abuse limits; fault/security/malicious-client tests. Human-only coded rooms + quick join.
- **P5 - Production art/UI/audio/perf:** conditional generator lane (credentials permitting; Grok /imagine route), ship/environment kits, full UI state matrix (Emil-skills pass on HUD/menus), licensed music, adaptive quality ladder, payload gates, device budget verification.
- **P6 - Release proof:** full bot/visual/network soak, photosensitivity + accessibility audit, license/secret/schema-migration checks, scorecard + fresh-eyes review, release notes disclosing casual movement authority + desktop-only target.

Critical path: **feel → energy/combat → AI → track 2 → match-scoped multiplayer → production/perf → release.** N0 is a safety gate beside G0, not permission to pull P4 forward.
